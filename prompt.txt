Our customer, Xome, is looking to build an Agentic Email & Personalized Communication Platform for marketing campaigns, powered by user behavior and property data.

⸻

Context

Xome is an online real estate marketplace where users browse, bid, and interact with property listings and auctions. The platform captures rich data such as:

* user_id
* listing_id, property_id
* Property attributes (price, location, type, propert type (house/condo/single family etc...), number of bedroom, number of bathroom, school rating etc.)
* User location (city/state)
* Auction details (event name, auction date)
* User browsing behavior (views, clicks, bids, favorites)

⸻

Important Data Distinction

There are two key types of input datasets. Please include the other datasets as needed.

1. User Browsing / Activity Tables (Context Only)

* Capture user engagement history:
    * Browsed properties
    * Viewed listings
    * Clicked/favorited properties
* Used ONLY for:
    * Understanding user intent
    * Personalization context for messaging
* ❌ Not used for selecting campaign properties

⸻

2. Recommendation Output Table (PRIMARY SOURCE FOR CAMPAIGNS)

* Generated by an existing recommendation engine/model
* Contains:
    * user_id
    * property_id
    * recommendation_score
* This table defines:
    * Which properties are eligible for outreach
    * Ranking for email campaigns

⚠️ Critical Rule:

* Email campaigns must ONLY use properties from the recommendation output table
* Browsing data cannot influence which properties are selected—only how the message is personalized

⸻

Business Objective (IMPORTANT)

The Agentic Email & Personalized Communication Platform is designed specifically for marketing campaigns.

Goal of the system:

* Send personalized campaign emails to targeted users
* Promote recommended properties to users with high intent
* Enable property sellers to get exposure and maximize chances of selling their properties
* Improve conversion by matching users with highly relevant recommended listings

👉 In short:
This system helps market and sell properties by reaching the right users at the right time using AI-driven personalization

Part 2: Agentic Campaign Platform (Databricks Apps with LangGraph)

The customer wants to build an Agentic Email Campaign Platform using Databricks Apps, leveraging the existing LangGraph-based agent template as the core orchestration framework.

⸻

Application Behavior (Campaign-Focused)

* The application is used by marketing teams to run property promotion campaigns
* The goal is to:
    * Select a target user
    * Select a location filter (city/state)
    * Generate a personalized campaign email promoting properties

⸻

How it works

* Candidate properties are retrieved ONLY from the Recommendation Output Table
* These properties represent:
    * Precomputed best matches between users and properties
    * Campaign-eligible listings for outreach
* The system selects Top 5 properties based on recommendation_score
* An AI agent (LangGraph + Sonnet 4.6) generates a persuasive campaign email that:
    * Promotes these properties to the user
    * Encourages engagement and purchase intent
    * Helps sellers market their properties effectively
* User browsing/activity data is used only for:
    * Personalization context
    * Messaging tone and relevance
    * NOT for selection or ranking

⸻

Technical Requirements

* Use Databricks Foundation Model (Sonnet 4.6) for:
    * Campaign email generation
    * Reasoning and personalization
    * Narrative construction
* Use LangGraph as the primary orchestration framework
    * Genie is implemented as a tool inside LangGraph
    * Databricks SDK is used inside tools for:
        * Data access
        * Query execution
        * Model invocation
* Architecture must be:
    * Modular
    * Extensible (future tools: segmentation, A/B testing, campaign optimization)
    * Production-ready for marketing campaign workloads
